A abordagem visa criar emparelhamentos que levem em consideração as preferências dos alunos em relação aos projetos, bem como uma nota de corte 
específica para cada projeto. Seguem as principais características do código:

1.Um projeto é selecionado entre os disponíveis, e suas vagas e nota de corte são determinadas.
2.Alunos que preferem o projeto atual e atendem à nota de corte são identificados e ordenados com base na nota e preferência.
3.Um aluno é emparelhado com o projeto atual, considerando a ordem de preferência e garantindo que o emparelhamento seja estável

A implementação aborda a estabilidade nos emparelhamentos, considerando a preferência dos alunos e a nota de corte dos projetos.
